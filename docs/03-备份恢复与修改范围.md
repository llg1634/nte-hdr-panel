# 备份恢复与修改范围

## 修改范围

工具会写入：

```text
%LOCALAPPDATA%\HT\Saved\Config\Windows\Engine.ini
```

工具可能读写 NVIDIA HUD 注册表：

```text
HKLM\SOFTWARE\NVIDIA Corporation\Global\NGXCore\ShowDlssIndicator
```

HUD 注册表只控制 HUD 显示，不控制 HDR 本身。

## 备份位置

每次写入都会创建：

```text
%LOCALAPPDATA%\HT\Saved\Config\Windows\_nte_hdr_backups\<时间戳>
```

备份内容：

```text
manifest.json
Engine.ini
```

如果写入前 `Engine.ini` 不存在，manifest 会记录不存在，恢复时删除工具创建的文件。

## 恢复逻辑

恢复目标是回到那次写入之前的状态：

如果写入前存在原文件，恢复时复制回原文件。  
如果写入前不存在原文件，恢复时删除当前 `Engine.ini`。  
如果写入前原文件只读，恢复后保持只读。  
如果写入前原文件可写，恢复后保持可写。

## 原版回退

如果你不是想恢复某一次 HDR 参数，而是想回到“未配置 HDR”的原始状态，面板会额外识别一份固定候选：

- 最早一份 `Engine.ini` 已存在的备份
- 该备份本身不包含 HDR 参数
- 该备份类型不是 `hdr-config`

UI 里的：

```text
恢复未配置 HDR 的原版
```

就是固定恢复这份候选，不依赖下拉框选择。

## 建议

不要手动删除 `_nte_hdr_backups`，除非你确认不需要恢复历史状态。
